# Промпты и процесс разработки

## Основные этапы разработки

### 1. Инициализация проекта
**Запрос:** Создать структуру Python CLI для работы с arXiv  
**Результат:** Каркас проекта с папками `arxiv_cli/{commands,api,utils}`, `tests/`

### 2. Стратегия реализации
**Запрос:** Определить порядок реализации функций  
**Решение:** Инкрементный подход — от простого к сложному:
1. `download` (базовая, быстрый результат)
2. `search` (основная функциональность)
3. `export` (работа с метаданными)
4. `list` (управление библиотекой)
5. `track` (отслеживание версий)
6. `digest` (агрегация данных)
7. `subscribe` (подписки с состоянием)

### 3. Реализация базовой инфраструктуры
**Задачи:**
- API client с HTTP-запросами к arXiv
- Parser для Atom XML (feedparser)
- Обработка ошибок (ArxivAPIError)

### 4. Команда download
**Функции:**
- Скачивание по ID
- Автоименование `{id}_{author}_{year}.pdf`
- Пакетное скачивание из файла
- Сохранение метаданных в библиотеку

### 5. Команда search
**Функции:**
- Построение сложных запросов (all:, cat:, au:, ti:)
- Пагинация (--page, --per-page)
- Фильтр по датам (--from, --to)
- Форматы: text, json, table, compact

### 6. Библиотека статей (library.py)
**Расширения:**
- Статусы: read/unread
- Флаг starred
- Теги
- Полнотекстовый поиск
- Сортировка

### 7. Команды управления (add, remove, info, list)
**Задачи:**
- Добавление без скачивания PDF
- Удаление с подтверждением
- Детальная информация
- Управление тегами

### 8. Команда export
**Форматы:**
- BibTeX с правильными полями
- CSL JSON для Zotero/Mendeley
- Фильтры по категориям и тегам

### 9. Команда track
**Отслеживание версий:**
- История версий статей
- Автоматическая проверка обновлений
- Сравнение дат updated

### 10. Команда digest
**Дайджесты:**
- Периоды: day/week/month
- Группировка по категориям
- Статистика
- Экспорт в Markdown

### 11. Команда subscribe
**Подписки:**
- Сохранение запросов
- Детекция новых статей (сравнение ID)
- Массовая проверка

### 12. Rate limit handling
**Проблема:** arXiv API возвращает 429 при частых запросах  
**Решение:** Автоматические повторы с экспоненциальной задержкой

### 13. Тестирование
**Итерации:**
- Первая волна: 30 тестов (24% покрытие)
- Вторая волна: 65 тестов (32% покрытие)
- Третья волна: 96 тестов (50% покрытие)
- Финал: 117 тестов (75% покрытие)

## Ключевые решения

### Архитектурные:
- **Модульная структура** — разделение на api, commands, utils
- **Библиотека статей** — JSON хранилище `~/.arxiv-cli/library.json`
- **Подписки** — отдельный файл `subscriptions.json`
- **Click framework** — удобный CLI с подкомандами

### Технические:
- **feedparser** для парсинга Atom XML
- **requests** с retry logic
- **Автоматическое именование** файлов с метаданными
- **Фильтры и сортировка** на стороне клиента
- **Версионирование** через сравнение поля `updated`

### Качество кода:
- **Type hints** в docstrings
- **Error handling** с понятными сообщениями
- **Verbose mode** для отладки
- **Comprehensive tests** — unit + integration

## Время разработки

**Примерное распределение:**
- Структура + инфраструктура: ~30 минут
- download + search: ~30 минут
- export + list + управление: ~40 минут
- track + digest + subscribe: ~60 минут
- Rate limit handling: ~20 минут
- Тестирование (3 итерации): ~90 минут
- Документация: ~30 минут

**Итого:** ~5 часов активной работы
