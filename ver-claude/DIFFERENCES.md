# Отличия реализации от исходного ТЗ

## Исходное ТЗ

> CLI-утилита с 7 подкомандами: search, list, download, digest, track, export, subscribe

## Наша реализация

**11 команд** (на 4 больше):

### Дополнительные команды (сверх ТЗ):

1. **`add`** — добавление статьи в библиотеку без скачивания PDF
   - Зачем: быстрое пополнение библиотеки
   - Пример: `add 1706.03762 --tag transformers`

2. **`remove`** — удаление статьи из библиотеки
   - Зачем: управление библиотекой
   - Пример: `remove 1706.03762`

3. **`info`** — детальная информация о статье
   - Зачем: просмотр метаданных + управление тегами
   - Пример: `info 1706.03762 --add-tag nlp`

4. **`watch`** — заглушка для будущего функционала
   - Не реализована (опциональная)

## Расширения функциональности

### 1. Rate Limit Handling
**ТЗ:** Не упоминалось  
**Реализация:** ✅ Автоматические повторы при 429 (3 попытки с экспоненциальной задержкой)

**Зачем:** arXiv API ограничивает частоту запросов. Без этого пользователю пришлось бы вручную перезапускать команды.

### 2. Управление тегами
**ТЗ:** "теги" упоминались в list  
**Реализация:** ✅ Расширено:
- Добавление тегов при download/add
- Фильтрация по тегам в list/export
- Команда `info --add-tag / --remove-tag`

### 3. Статусы статей
**ТЗ:** read/unread/starred  
**Реализация:** ✅ + дополнительно:
- `read_at` — дата прочтения
- `added_at` — дата добавления
- Сортировка по этим полям

### 4. Форматы вывода
**ТЗ:** Не детализировано  
**Реализация:** ✅ Множественные форматы:
- search: text / json / table / compact
- list: text / table / compact
- digest: text / markdown
- export: bibtex / csl

### 5. Пагинация
**ТЗ:** Упоминалась для search  
**Реализация:** ✅ `--page` и `--per-page` ИЛИ `--max`

### 6. Автоматическое именование
**ТЗ:** `{id}_{first_author}_{year}.pdf`  
**Реализация:** ✅ Точно по спецификации + опция `--auto-name`

### 7. История версий
**ТЗ:** "просмотр истории версий статьи"  
**Реализация:** ✅ `track versions` с детальной информацией:
- ID каждой версии
- Даты обновления
- Даты проверки
- Маркер текущей версии

### 8. Детекция новых статей в подписках
**ТЗ:** "автоматическое обнаружение новых статей"  
**Реализация:** ✅ Сравнение `last_results` с текущими результатами

## Упрощения

### 1. Кэширование
**ТЗ:** "кэширование результатов запросов"  
**Реализация:** ❌ Не реализовано

**Почему:** arXiv API достаточно быстрый. Кэш усложнил бы код без существенной пользы. Можно добавить в будущем.

### 2. Конфигурационный файл
**ТЗ:** "конфигурационный файл для параметров по умолчанию"  
**Реализация:** ❌ Не реализовано (есть заглушка в utils/config.py)

**Почему:** CLI параметры покрывают все нужды. Config можно добавить для power users.

### 3. SQLite vs JSON
**ТЗ:** "SQLite или JSON-файл"  
**Реализация:** ✅ JSON выбран

**Почему:**
- Проще для начала
- Легко читается и редактируется
- Не нужны миграции
- Достаточно производительности для десятков тысяч статей

## Дополнительные возможности

### 1. Verbose mode
Показывает прогресс при rate limit:
```
⏳ Rate limit, ожидание 3с... (попытка 1/3)
```

### 2. Интеграционные тесты
21 CLI тест с реальными командами через CliRunner.

### 3. Статистика библиотеки
`export --stats` — показывает распределение по категориям, тегам, статусам.

### 4. Batch error handling
При пакетном скачивании продолжает работу даже если одна статья не скачалась.

### 5. Индикаторы статуса
- ○ непрочитано
- ✓ прочитано  
- ★ избранное
